# Go-To-Market Checklist (v1)

## Product readiness

1. README quickstart commands validated against current code.
2. Showcase gallery assets refreshed from deterministic capture pipeline.
3. Trust verification commands tested on clean checkout.
4. Release notes include exact verify steps and expected outputs.

## Brand and presence

1. Final product logo and social preview image prepared.
2. GitHub repository metadata aligned (description, topics, homepage).
3. Public showcase page published from docs/static hosting.
4. Consistent naming across repo, website, and social accounts.

## Launch assets

1. One hero screenshot from showcase SVG assets.
2. One 45-90 second terminal walkthrough clip.
3. One deterministic proof demo snippet (`viewmodel.hash` rerun match).
4. One share-safe refusal demo snippet (`refusal-report.json` flow).

## Distribution

1. Tagged release with changelog and verification section.
2. Release artifacts and checksum instructions published.
3. Copy-paste install/run instructions for Linux and macOS.
4. Issue templates and support channels verified.

## Community motion

1. Product Hunt listing draft prepared.
2. Launch post draft prepared for X/LinkedIn/Reddit/HN.
3. FAQ answers prepared for determinism, safety, and roadmap.
4. Post-launch triage owner and response SLA defined.

## Post-launch first 14 days

1. Collect top onboarding friction points from issues/discussions.
2. Track activation signal: first successful local replay verification.
3. Track proof signal: users posting matching hash reruns.
4. Ship first polish patch focused on onboarding friction.
